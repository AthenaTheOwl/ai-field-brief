# Lens: Adoption Action

Identify practical adoption candidates.

Allowed action surfaces:
- prompt
- config
- eval
- workflow
- agent role
- tool policy
- runtime adapter
- source registry
- architecture note
- experiment
- creative OS
- software control plane
- personal knowledge base
- watchlist

Each action must include a test plan, expected benefit, risk, effort, and disposition.
Do not force an action when the source is merely interesting.
